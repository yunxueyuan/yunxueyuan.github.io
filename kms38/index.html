    KMS38激活

    概述

    • 如何使用？请在此处找到说明。
    • 此激活方法激活Windows 10/11和Windows Server（14393及更高版本），直到2038年1月19日03:14:07 UTC。
    • 此激活方法不会在系统上存储任何文件。
    • 此激活方法可以离线使用。
    • 与HWID和在线KMS激活选项不同，KMS38与任何官方激活方法都不匹配。

    它是如何工作的？

    • 在真实的KMS激活环境中，激活最长持续时间为180天。这是使用有效的许可证和服务器设置完成的。

    • 在Windows的主要升级过程中，系统使用gatherosstate.exe传递剩余的KMS激活期。它通过创建一个可以离线使用的票证来完成。

    • 诀窍是，我们可以欺骗gatherosstate.exe关于剩余的KMS激活期，并手动将所需的期限最多设置为2038年1月19日03:14:07 UTC。

    • 为什么它的有效期限制为2038年？ 这与Y2K38问题有关，因为这个日期（2038年1月19日03:14:07 UTC）是我们可以向gatherosstate.exe提供的最大日期，否则它会循环回到1970年。

    • 如何欺骗gatherosstate.exe？ 有两种方法。

      1- 在gatherosstate.exe旁边放置一个自定义的slc.dll文件：

      gatherosstate.exe使用系统的C:\Windows\System32\slc.dll文件收集系统信息。如果我们在gatherosstate.exe旁边放置一个自定义的slc.dll文件，可以向它发送垃圾数据，那么它将简单地接受它并生成有效的票据，允许激活直到2038年1月19日03:14:07 UTC。

      2- 修改gatherosstate.exe文件本身，使其不检查系统的激活状态，我们可以随心所欲地设置激活期限。

    • 注：

      • 明确一点，我们不是修改/修补任何板载系统文件以获取票证。gatherosstate.exe是ISO文件的一部分，不在C驱动器系统文件中。系统的slc.dll文件没有被修改，我们只是在生成票证的短暂过程中使用自定义的slc.dll文件。

      • 最新的MAS没有使用这些方法，而是使用现成的通用票据（有关手动激活信息，请参见下文）。

    问：Microsoft是否可以阻止这种类型的激活？

    答：不能直接阻止。他们只能更新Clipup，允许最长激活期为180天。除此之外，他们无法采取其他行动。票据根本没有发送给Microsoft，因此他们无法直接阻止它们或采取行动。

    • 任何Windows的评估版本（即“EVAL”LTSB / C）都无法激活。

    如何删除KMS38？

    • 在MAS中，进入KMS38激活并应用“Remove KM38 Protection”选项。
    • 然后，在MAS中，进入“故障排除”并应用“修复许可证”选项。
    • 完成。

    KMS38-服务器Cor / Acor

    • Windows Server Cor / Acor（无GUI）版本没有clipup.exe文件。

    • 要进行KMS38激活，您需要从此链接下载ClipUp.exe文件。

      文件：ClipUp.exe SHA-256：0d6e9f6bbd0321eda149658d96040cb4f79e0bd93ba60061f25b28fecbf4d4ef

    • 此文件具有可验证的数字签名。您也可以从官方的Windows Server 2016 x64 RTM ISO获取此文件。

    • 将ClipUp.exe放在KMS38激活脚本旁边。那将是MAS_AIO.cmd或KMS38_Activation.cmd

    • 激活脚本将在当前文件夹（从脚本运行的位置）中检查ClipUp.exe并相应地使用它。

    KMS38保护

    • KMS38激活容易被180天KMS激活器意外替换。

    • 从MAS 1.7开始，脚本将默认启用KMS38保护。

    • 如果您要使用abbodi1406的KMS_VL_ALL或MAS（在线KMS选项）进行Office激活，则可以选择删除此保护。

    • 以下是脚本如何保护KMS38的说明。

    • 在KMS激活方法中，Windows首先检查注册为特定KMS的KMS IP，如果未定义，则检查全局KMS IP。另一个事实是，如果在Windows 8及更高版本的操作系统中将Localhost（IP范围127.0.0.0/8）定义为KMS IP，则Windows将不接受它作为有效的KMS IP。该脚本仅利用上述事实保护KMS38激活不会被任何其他“普通”180天KMS激活替换，同时可以使用该KMS激活器进行Office激活。

    • 脚本步骤-

      • 将Windows版本特定的KMS IP设置为Localhost（127.0.0.2）

      • 保护以下注册表不受编辑和删除

      • 完成。

    手动激活

    这是给那些想要进行手动激活的人。如果你想要一个工具来帮你完成这个过程，请点击这里。

    我们可以将手动激活过程分为两个部分。

    1- 使用预制凭证

    • 以管理员身份打开Windows Powershell，并按照给出的顺序输入以下列出的命令。

    • 使用以下命令输入密钥（将替换为上述列表中的密钥）：

      slmgr /ipk <key>

    • 从这里下载Universal ticket，并解压下载的文件。

    • 在解压后的文件夹中找到名为KMS.xml的文件。

    • 将凭证文件复制并粘贴到以下文件夹中：

      C:\ProgramData\Microsoft\Windows\ClipSVC\GenuineTicket

    • 现在在PowerShell中运行以下命令以应用凭证：

      clipup -v -o

    • 使用以下命令检查激活状态

      slmgr /xpr

    • 完成。

    2- 从零开始

    在这个过程中，我们将从零开始进行激活。这是基于Universal ticket方法的。在这里，我们将创建与MAS KMS38脚本中使用的相同的凭证，并用它来激活系统。

    • 从以下官方微软链接下载文件并提取此.cab文件。

      https://download.microsoft.com/download/9/A/E/9AE69DD5-BA93-44E0-864E-180F5E700AB4/adk/Installers/14f4df8a2a7fc82a4f415cf6a341415d.cab

    • 找到名为filf8377e82b29deadca67bc4858ed3fba9的文件，并将其重命名为gatherosstate.exe

    • 在C驱动器上创建一个名为Files的文件夹，C:\Files，并将gatherosstate.exe文件复制到该文件夹中。

    • 以管理员身份打开Windows Powershell，并按照给出的顺序输入以下列出的命令。
