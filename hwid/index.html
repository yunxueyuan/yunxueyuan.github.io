    HWID数字权利激活

    概述

    • 如何使用？请在此处查找说明。

    • 此激活仅支持Windows 10/11。

    • 此激活不会存储或修改/修补系统中的任何文件。

    • 此激活是您系统硬件的永久激活。

    • 在一个系统上，此激活可以为所有支持的版本创建，并且所有版本都可以链接到Microsoft帐户，不会出现任何问题。

    • 一旦系统被激活，此激活将无法被删除，因为许可证存储在Microsoft服务器中，而不是用户的系统中。MS检查硬件ID（HWID），如果在其数据库中发现许可证，则系统将自动激活。这就是官方数字许可证的工作原理。

    • 硬件发生重大更改（例如主板）可能会使系统失活。如果您的激活已链接到在线Microsoft帐户，则可以重新激活因重大硬件更改而失活的系统。

    • 为了成功激活，必须启用Internet连接。如果您试图在不满足这些条件的情况下进行激活，则在满足条件时，系统将自动激活。

    • Windows重新安装后的自动激活场景：

      • 需要Internet连接。（仅在激活时）

      • 如果使用零售（消费者）版进行安装，则系统将自动激活。

      • 如果使用VL（商业）版进行安装，则系统将不会自动激活。在这种情况下，如果用户不想再使用此脚本重新激活，则用户将必须插入该Windows版本的零售/OEM密钥（请在本页下方查找密钥）以激活。

    它是如何工作的？

    • 在从Windows 7升级到Windows 10的官方升级过程中，Microsoft为Windows 10提供了HWID（数字许可证）激活，无需任何费用。

    • 在后台，升级过程运行一个名为gatherosstate.exe（在Windows 10/11 ISO中可用）的文件，并检查当前Windows的许可证是否已激活，如果发现已激活，则生成一个有效的GenuineTicket.xml票证，该票证发送到Microsoft，而MS则授权一项许可证。

    • 因此，如果我们只是欺骗gatherosstate.exe文件，使其认为生成票证的条件已满足，它将生成一个有效的票证，可用于获取有效的HWID（数字许可证）。

    • 如何欺骗gatherosstate.exe？ 有两种方法。

      1-在gatherosstate.exe旁边放置自定义slc.dll文件：

      gatherosstate.exe使用系统的C:\Windows\System32\slc.dll文件来收集系统信息。如果我们在gatherosstate.exe旁边放置一个自定义的slc.dll文件，它可以将垃圾数据发送到gatherosstate.exe，然后它将简单地接受它并生成有效的许可证。

      2- 修改gatherosstate.exe文件本身，使其不检查系统的激活状态，可以直接创建有效的许 可证。

    • 注：

      • 需要明确的是，我们不会修改/打补丁任何内置系统文件以获得数字许可证。gatherosstate.exe是ISO文件的一部分，不可在C驱动器的系统文件中找到。系统的slc.dll文件没有被修改，而是仅在生成许可证的瞬间使用自定义的slc.dll。

      • 如果您想了解有关这两种方法的更多信息，请查看此存储库MAS-Legacy-Methods。

      • 最新的MAS不使用这些方法之一，而是使用现成的通用许可证（请查看下面的信息）。

    票据类型

    有许多生成票据的方法，主要可以将票据类型分为三类。

    1. 下级票据 - 这是最简单的票据生成过程。在此方法中，系统使用下级密钥签署票据，票据的 OSVersion =<10。如果在 Windows 10/11 上生成，则 MS 理想上不应该授予激活，但他们确实授予了，并且该方法从过去的 4-5 年中一直在持续工作。几乎每个 HWID 激活工具（在撰写本文时）都基于此方法。
    2. 锁匣票据 - 这是稍微棘手的方法。票据由系统使用锁匣密钥签署，票据的 OSVersion 为 10。此票据生成过程涉及清除系统中的 Clipsvc 许可证和 IdentityCRL 注册表键。由于过程中出现了一些问题，该方法不是主要在 MAS 中使用的方法，但这些类型的票据与在已激活的 Windows 10/11 上系统生成的真正票据相同。
    3. 通用票据 - 在此方法中，我们将票据中的 OSVersion 设为 =<5，并在应用票据时，由于较低的 OSVersion，系统会忽略票据中的 HWID 并授予激活。此票据可应用于任何系统。这些票据可以是下级票据或锁匣票据。MS 理想上不应该授予激活，但他们确实授予了。

    MAS 的最新版本使用通用票据方法。其他票据方法脚本可在此存储库 MAS-Legacy-Methods 中找到。

    现在一个问题，微软是否可以阻止新的请求或撤销已经建立的 HWID 许可证？

    • 撤销许可证将是过于极端的举措，并会面临许多复杂性和风险，可能会使有效许可证失效。然而，也许他们可以阻止来自下级和通用票据的新激活请求。但这些工具已经工作了 4-5 年，而他们似乎并不太在意消费者盗版；这不是他们获得最多收益的地方。

    如何删除HWID？

    • 您无法删除它，就像您无法删除官方Windows数字（HWID）许可证一样。一旦系统激活，此激活无法删除，因为许可证存储在Microsoft服务器中而不是用户的系统中。 Microsoft检查硬件ID（HWID），如果在其数据库中找到许可证，则系统将自动激活。

    手动激活

    这是为那些想要执行手动激活的人提供的。如果您想要一个工具来为您执行此操作，请在此处检查。

    我们可以将手动激活过程分为两个部分。

    1-使用现成的票证

    • 确保互联网已启用。

    • 以管理员身份打开Windows Powershell，并按照给定的顺序输入以下列出的命令。

    • 使用以下命令输入密钥（用上面列表中的密钥替换<key>）

      slmgr /ipk <key>

    • 从此处下载通用票证并提取下载的文件。

    • 现在在Powershell中输入下面的代码

      (Get-ItemProperty HKLM:\SYSTEM\CurrentControlSet\Control\ProductOptions).OSProductPfn

    • 此命令将向您显示一些文本，例如Microsoft.Windows.48.X19-98841_8wekyb3d8bbwe

    • 您需要在先前提取的文件夹中找到完全相同的名称票证文件。

    • 将该票证文件复制并粘贴到以下文件夹中

      C:\ProgramData\Microsoft\Windows\ClipSVC\GenuineTicket

    • 现在在Powershell中运行以下命令以应用票证

      clipup -v -o

    • 使用以下命令激活Windows

      slmgr /ato

    • 使用以下命令检查激活状态

      slmgr /xpr

    • 完成。

    2-从头开始

    在此过程中，我们将从头开始执行激活。这是基于通用票证方法的。在这里，我们将创建与MAS HWID脚本中使用的相同的票证，并用它来激活系统。
