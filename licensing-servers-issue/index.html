    许可证服务器问题

    • 在某些情况下，由于系统无法连接到HWID许可证服务器，HWID激活可能会失败。
    • 如果是这种情况，激活脚本会通知您。我们建议只在脚本告诉您这样做时才按照下面的指南进行操作。如果您不确定，可以在此处寻求帮助。
    • 这可能主要是由于DNS或MS许可证服务器本身的问题或系统中的防火墙规则引起的。以下是其解决方法：
    • 选择哪种方法来修复？
      • 如果您位于中国，请选择修改主机文件选项。
      • 如果您不在中国，请先尝试修复DNS选项，如果不起作用，请尝试VPN方法或尝试另一个Internet连接。
      • 如果您仍然遇到问题，请在此处检查。

    修复DNS

    • 从这里下载Fix_DNS.zip文件。
    • 解压缩zip文件，右键单击Fix_DNS.cmd文件并以管理员身份运行它。
    • 按照屏幕上的说明操作。
    • 完成后，请尝试HWID激活。

    VPN

    • 如果脚本显示无法连接到许可证服务器，则可以安装以下任一VPN来修复此问题。
    • Cloudflare WARP- https://1.1.1.1/
    • Psiphon - https://psiphon.ca/
    • 完成后，请尝试HWID激活。

    修改主机文件

    在2023年2月8日左右，中国的MS许可证服务器无法正确连接，这扰乱了许多Windows服务，包括激活。解决此问题的方法是在主机文件中硬编码另一个国家的MS许可证服务器IP，Windows将使用该IP进行激活。如果由于任何原因DNS无法解析MS许可证服务器，则也可以使用此方法。

    • 以管理员身份打开PowerShell或终端（不要打开CMD）。
    • 复制下面的行并按Enter：
    $filePath = "C:\Windows\System32\drivers\etc\hosts"

Set-ItemProperty -Path $filePath -Name IsReadOnly -Value $false

Set-Content -Path $filePath -Value (Get-Content $filePath | ForEach-Object { $_ -replace "124.108.22.138 licensing.mp.microsoft.com", "" })

Add-Content -Path $filePath -Value "124.108.22.138 licensing.mp.microsoft.com"

Set-ItemProperty -Path $filePath -Name IsReadOnly -Value $true
    • 完成。

    现在尝试 HWID 激活。 要恢复它，请运行以下命令。

    $filePath = "C:\Windows\System32\drivers\etc\hosts"

Set-ItemProperty -Path $filePath -Name IsReadOnly -Value $false

Set-Content -Path $filePath -Value (Get-Content $filePath | ForEach-Object { $_ -replace "124.108.22.138 licensing.mp.microsoft.com", "" })

Set-ItemProperty -Path $filePath -Name IsReadOnly -Value $true
