    $OEM$ 文件夹

    （Windows 预激活）

    概述

    • 要创建带有预激活的 Windows 安装 .iso，执行以下操作：
      • 使用 MAS 脚本将 $OEM$ 文件夹提取到桌面上。
      • 将 $OEM$ 文件夹复制到 Windows 安装介质（.iso 或 USB）中的 sources 文件夹中。
      • 在修改后的 .iso 或可引导的 USB 驱动器中，目录将显示为 \sources\$OEM$。
      • 现在使用此 .iso 或可引导的 USB 驱动器安装 Windows，它将在启动时自动激活（KMS38），或在第一次连接互联网时自动激活（HWID 或在线 KMS）。
      • 您可以在此处查看激活方法的详细信息，并根据您的需求选择激活方法。
      • 在 Windows 8 及更高版本中，如果版本的默认安装密钥为 OEM 渠道，则禁用运行 setupcomplete.cmd。例如- 对于 Windows 10 Iot Enterprise，预激活将无法工作，因为安装的密钥属于 OEM 渠道。

    KMS38-Server Cor/Acor

    • Windows Server Cor/Acor（无 GUI）版本没有 clipup.exe 文件。

    • 要进行 KMS38 激活，您需要从此链接下载 ClipUp.exe 文件。

      文件：ClipUp.exe

      SHA-256：0d6e9f6bbd0321eda149658d96040cb4f79e0bd93ba60061f25b28fecbf4d4ef

    • 此文件具有可验证的数字签名。您还可以从官方 Windows Server 2016 x64 RTM ISO 获取此文件。

    • 将 ClipUp.exe 放置在 KMS38 激活脚本旁边。这将是 MAS_AIO.cmd 或 KMS38_Activation.cmd。

    • 激活脚本将在当前文件夹中检查 ClipUp.exe（从脚本正在运行的位置），并相应地使用它。

    HWID

    • 使用 HWID 激活时，不会在系统上存储任何文件，而 Windows 10-11 在第一次连接互联网时激活。

    KMS38

    • 使用 KMS38 激活时，不会在系统上存储任何文件，而 Windows 10-11-Server 将立即激活，无需互联网。

    在线 KMS

    • 使用在线 KMS 激活时，Windows 和 Office 都将在第一次连接互联网时激活。此选项使用续订任务进行终身激活。

    HWID + 在线 KMS

    • 使用此方法，Windows 10-11 将通过 HWID 激活，而 Office 将使用在线 KMS 激活。

    KMS38 + 在线 KMS

    • 使用此方法，Windows 10-11-Server 将通过 KMS38 激活，而 Office 将使用在线 KMS 激活。
